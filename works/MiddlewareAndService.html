Middleware and Service


Service Oriented Architecture


In this course we have been introduced to the web service. We first learned how to implement soap type web services with a bottom-up approach. We used a Tomcat server to deploy the achievements. We also saw some notions on WSDL before using the SpringBoot framework to automate the development and deployment of web services using the REST API.
Secondly, we worked on a project which consisted of an IoT platform to automate the management of INSA's room. We first used Jira for project management. It allows you to split the project into several sprints. We used oneM2M to simulate the sensors that will be in the classroom (Movement, interior temperature, exterior temperature). We also used SpringBoot to retrieve and process data from simulated sensors. After our web service was ready, we performed Junit tests to make sure it worked. We finally created an "interface" to visualize the state and the value of all the sensors.

Middleware for the Internet of Things


Throughout this course, we have studied 4 different IoT tools.
We started with the MQTT protocol. It is a TCP / IP-based publish and subscribe protocol. We have developed a simple application with an ESP8266 using MQTT as the communication protocol with the server. The application consisted of turning an LED on and off.
Subsequently, we used the mangOH yellow cards from Sierra Wireless. We have discovered and explored its Graphical User Interface and some of its features.
The OneM2M standard was our next subject of study. We discovered and used this standard through REST requests to create, modify and delete an IoT resource tree. We played with it using multiple REST requests generated from POSTMAN and created sensors and recorded their data / states.
Finally, we used Node-Red in order to link the tools seen previously. We used Node-Red to link OneM2M to MQTT.

Skills developped


Service Oriented Architecture

Know how to define a Service Oriented Architecture (4)

We followed a MOOC course and we discover several standards and technology used in IoT. We learn the basic notions of IoT and then we went into more detail about the oneM2M standard.

Deploy an SOA with web services (4)

We deploy a service oriented architecture that contains web services using Spring Boot and the REST API.

Deploy and configure an SOA using SOAP (4)

During pratical works, we configured, thanks to a WSDL file, and deployed web services using SOAP protocol on a Tomcat Server.

Deploy and configure an SOA using REST (4)

During the practical work, we created REST webservice and deployed it on a Tomcat Server. During the project, we also used the REST API and we deployed our application using SpringBoot.

Integrate a process manager in an SOA

We did not integrate a process manager in an SOA neither during the project nor during the pratical works.


Middleware for the Internet of Things

Know how to situate the main standards for the Internet of Things (4)

We took a MOOC which introduced us to the main standards of the Internet of Things. We then explored in more depth the OneM2M standards and the MQTT protocol.

Deploy an architecture compliant to an IoT standard and omplement a sensor network (4)

In this course, we deployed a connected object architecture using the oneM2M standard and the REST API. We made it so that these objects can communicate with each other.

Deploy and configure and IoT architecture using OM2M (4)

During the project, we implemented on OM2M a simulated IoT sensor architecture using the oneM2M standard.

Interact with the different resources of the architecture using REST services (4)

We used the REST API to implement the sensor architecture.To communicate with the different resources of this architecture, we also used REST service to access specific Content Instances (Sensor data).

Integrate a new technology into the deployed architecture (4)

In the pratical work, we extended the MQTT and oneM2M standards through the intermediary of Node-red so that the reception of data on MQTT topics have an influence on the oneM2M architecture.


Adaptability : Cloud and Autonomic Computing

Understand the concept of cloud computing (4)

Throughout the Labs, we saw several notions of cloud computing (SaaS, IaaS and PaaS). We have also seen the principles of VMs and containers.

Use a IaaS-Type cloud service (4)

During the labs, we used an Infrastructure as a Service (IaaS). Through the use of Openstack, we have created several cloud environment instances

Deploy and adapt a cloud-based platform for IoT (4)

We have, during the practical work, realised several microservices. We used OpenStack to create a cloud environment so that each micro service is deployed on a different VM belonging to a network in order to create an interaction between the host and the micro services.
